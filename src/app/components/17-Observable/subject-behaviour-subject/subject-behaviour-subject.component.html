Subject, Behaviour Subject

Observable

in RxJS is a powerfull tool which streamlines async operations.
Obsevable is a base class for Subject, BehaviourSubject, AsyncSubject, ReplaySubject

Subject:

  • Subject is a multicast observable, meaning it can emit values to multiple subscribers simultaneously.
  • It does not hold any current value. If a subscriber joins after a value is emitted, the subscriber will not receive that value.
  • It emits values to all its subscribers as soon as it gets a new value.
  • Common Use Cases:
    • Emitting values triggered by user interactions (e.g., button clicks).
    • Streaming real-time data updates (e.g., from WebSocket or API calls).

BehaviorSubject:

  • BehaviorSubject is a multicast observable that stores the most recent value.
  • New subscribers always receive the last emitted value immediately upon subscribing, even if they subscribed after the value was emitted.
  • It emits new values to all subscribers as soon as it gets them.
  • Common Use Cases:
    • Storing application state (e.g., user authentication status).
    • Sharing data between parent and child components.
